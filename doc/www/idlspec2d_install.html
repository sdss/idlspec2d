SDSS IDLSPEC2D Installation

Installing IDLSPEC2D w/UPD

The following instructions are for installing IDLSPEC2D from the UPD repository at Fermilab. This would be the proper way to install a cut version of the code, though it appears that only the very first version of the code (v1_0) is in the repository.
  1. ``idl'' should already be installed on the system. Try the following:
    which idl
    which should return the path for idl. On some Fermi machines, you'll need to
    setup idl
  2. Install ``idlspec2d'', which will also install ``idlutils''. The following example is for installing version v1_0 (which is useless). 
    setup upd
upd install idlspec2d v1_0 -H Linux+2 "-c -f Linux+2"

Installing IDLSPEC2D from CVS w/SDSSTOOLS

The following instructions are for building a version of IDLSPEC2D from the current version of the code in the Fermilab CVS repository.
  1. ``idl'' should already be installed on the system. Try the following:
    which idl
    which should return the path for idl. On some Fermi machines, you'll need to
    setup idl
  2. Download the ``idlutils'' and ``idlspec2d'' packages. 
    cd
setup sdsscvs
cvs co idlutils
cvs co idlspec2d
    If these packages already exist, you may choose to update them instead: 
    cd ~/idlutils
cvs update -d
cd ~/idlspec2d
cvs update -d

  3. Log in as user ``products'' and declare the products: 
    su products    (need the password for this)
ups declare -f Linux+2 -m idlutils.table \
    -c -r $HOME/idlutils idlutils cvs
ups declare -f Linux+2 -m idlspec2d.table \
    -c -r $HOME/idlspec2d idlspec2d cvs
exit    (log out as user ``products'')
  4. Build the ``idlutils'' and ``idlspec2d'' packages. 
    setup sdsstools
setup idlutils
setup idlspec2d
cd ~/idlutils
sdssmake -idl -opt -debug
cd ~/idlspec2d
sdssmake -idl -opt -debug

The following instructions are for building a tagged version of IDLSPEC2D from the Fermilab CVS repository. In this example, we build version v4_1_2.
  1. ``idl'' should already be installed on the system. Try the following:
    which idl
    which should return the path for idl.

  2. Download the ``idlutils'' and ``idlspec2d'' packages, putting them in subdirectories named after the version. 
    setup sdsscvs
mkdir $HOME/idlutils $HOME/idlspec2d
setenv IDLVERS v4_1_2
cd $HOME/idlutils
cvs co -r $IDLVERS -d $IDLVERS idlutils
cd $HOME/idlspec2d
cvs co -r $IDLVERS -d $IDLVERS idlspec2d

  3. Log in as user ``products'' and declare this version of the code as current: 
    su products    (need the password for this)
ups declare -f Linux+2 -m idlutils.table \
    -c -r $HOME/idlutils/$IDLVERS/$IDLVERS idlutils $IDLVERS
ups declare -f Linux+2 -m idlspec2d.table \
    -c -r $HOME/idlspec2d/$IDLVERS/$IDLVERS idlspec2d $IDLVERS
exit    (log out as user ``products'')

  4. Build the ``idlutils'' and ``idlspec2d'' packages. 
    setup sdsstools
setup idlutils
setup idlspec2d
cd $HOME/idlutils/$IDLVERS
sdssmake -idl -opt -debug
cd $HOME/idlspec2d/$IDLVERS
sdssmake -idl -opt -debug

Installing IDLSPEC2D from CVS w/out SDSSTOOLS

The following instructions are for building a version of IDLSPEC2D without any other Fermi code (such as sdssmake). This is not the recommended method of installation.
  1. ``idl'' should already be installed on the system. Try the following:
    which idl
    which should return the path for idl.

  2. Download the ``idlutils'' and ``idlspec2d'' packages (as above).

  3. Build everything using the utility $IDLUTILS_DIR/bin/evilmake: 
    setenv IDLUTILS_DIR $HOME/idlutils
setenv IDLSPEC2D_DIR $HOME/idlspec2d
set path = ( $path {$IDLUTILS_DIR}/bin )
cd $IDLUTILS_DIR
evilmake all
cd $IDLSPEC2D_DIR
evilmake all

  4. Then set the necessary environment variables (assuming IDL version 5.2): 
    setenv IDL_DEVICE X 
    
setenv IDL_DIR /usr/local/rsi/idl_5.2
setenv IDL_PATH .:+$IDL_DIR/lib
setenv IDL_PATH {$IDL_PATH}:+{$IDLSPEC2D_DIR}/pro
setenv IDL_PATH {$IDL_PATH}:+{$IDLUTILS_DIR}/pro
setenv IDL_PATH {$IDL_PATH}:+{$IDLUTILS_DIR}/goddard/pro
    You may wish to add the above lines to your ``~/.cshrc'' file.
